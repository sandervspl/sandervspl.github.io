Interface

Een interface is een blauwdruk van een data type. Met een interface zeg je dus hoe een data type eruit moet zien.

Voorbeeld


                interface IPerson {
                  name: string;
                  age: number;
                  isAlive: boolean;
                  children: IPerson[];
                }

                const person: IPerson;
              

                // Error: Property 'children' is missing in type '{ name: string; age: number; isAlive: true; }'.
                const person: IPerson = {
                  name: 'Sander',
                  age: 27,
                  isAlive: true,
                };
Alles wat in een interface staat aangegeven is verplicht! Tenzij...

Optionals


                interface IPerson {
                  name: string;
                  age: number;
                  isAlive: boolean;
                  children?: IPerson[];
                }
              

                const person: IPerson = {
                  name: 'Sander',
                  age: 27,
                  isAlive: true,
                };

Geen errors!

Bonus: extenden


              interface IAthlete extends IPerson {
                sport: string;
              }
              
              // Vormt de volgende interface
              interface {
                name: string;
                age: number;
                isAlive: boolean;
                children?: IPerson[];
                sport: string;
              }

Een atleet is een persoon, maar een persoon is niet altijd een atleet!

Functions

Bij een function kan je aangeven wat de return value is.


              function hundred(): number {
                  return 100;
              }

              const thousand = (): number => {
                return 1000;
              }
            

              const helloWorld = (): string => {
                return 'Hello world!';
              }
Je kan ook de parameters een data type geven! 

              const talk = (text: string): string => {
                return `You say: ${text}`;
              }
            

              const personTalk = (person: IPerson, text: string): string => {
                return `${person.name} says: ${text}`;
              }

Maar... waarom?

In het kort: Het maakt het onderhouden van code, op lange termijn, makkelijker.

De voordelen

Type Annotations

  • Kan de code meer leesbaar maken
  • Kan het analyseren van code makkelijker maken
  • Maakt het refactoren van code makkelijker
  • Kan voor betere IDE support/experience zorgen
  • Kan eerder errors zien

(let op het woordje "kan")

Data Structures

Je kan data structuren makkelijk opstellen. Hierdoor is het makkelijk voor de programmeurs om te zien hoe een data structuur in elkaar zit. Denk bij data structuren aan

  • REST payload
  • config files
  • objects van, en naar, database

De nadelen

Natuurlijk is TS niet perfect!

Nadelen

  • Kan extra werk zijn om libraries/modules te installeren.
  • Extra boilerplate (meer code)
  • Kost dus (iets) meer tijd
